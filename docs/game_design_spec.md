# 企画・仕様書: Slay the Spire風 マルチプレイローグライクカードゲーム

## 1. プロジェクト概要
- **プラットフォーム**: ブラウザゲーム (将来的に VRChat / Udon# への移植も予定)
- **システムアーキテクチャ**: 
  1. **フロントエンド (View / UI)**: React / TypeScript
  2. **バックエンド (Network / Sync)**: C# ASP.NET Core API + SignalR (リアルタイムでの状態同期)
  3. **コアロジック (Domain)**: C# クラスライブラリ (表示やサーバ通信から完全に分離された純粋なゲームロジック群。将来Udon#にそのまま移植する基盤となる)
- **ジャンル**: 協力進行型 マルチプレイ・ローグライクカードゲーム (PvE協力メイン)

## 2. マッチング・ルーム管理
- ゲーム開始時に**シングルプレイ** または **マルチプレイ** を選択可能。
- マルチプレイ時はホストが合言葉（ルームコード）を設定してルーム（部屋）を作成し、他のプレイヤーは合言葉を入力してロビーに参加する。
- ロビーで全員の準備が完了した段階でゲーム（ダンジョン/階層）を開始する。

## 3. マップ移動・階層進行
- **扉のシステム**: プレイヤーの目の前に1〜3枚の「扉」が提示される。それぞれの扉には、その先に待ち受ける階層イベント（敵、宝箱、ランダムイベント、ショップ、休憩マス等）の内容が記されている。
- **移動の決定**: 
  - 全てのプレイヤーが進行する「扉」を選択した段階で決定される。
  - プレイヤー間で意見が割れる（異なる扉を選択した）場合は、**「抽選（ランダム）」**で進行先の扉が決定される。

## 4. バトルシステム (PvE)
### 4.1. 勝利条件・失敗条件
- **勝利条件**: 場に出現して出現している「敵カード」を全て倒すこと。（誰かが倒した時点で即座にその階層はクリア扱いとなる・早い者勝ち）
- **失敗(ゲームオーバー)条件**: 参加している全プレイヤーの体力が0になること。
- **死亡と復活**: 個人の体力が0になったプレイヤーは、その戦闘(階層)中は行動不能になり「無敵の観戦者」となる（自身のターン行動は行えず、敵からのダイレクト・ダメージも受け付けない）。戦闘のある階層をクリアした時点で、**「体力1を持った状態で自動復活」**する。

### 4.2. リソース管理
各プレイヤーが持つ以下のリソースは**「完全に独立したパーソナルリソース」**として管理され、パーティ間で共有されない。
- **デッキ**・**手札**・**捨て札**・**エナジー（行動力）**
- **ゴールド**(所持金)
- **レリック**(パッシブ効果アイテム)
※将来拡張: ショップで他プレイヤーの限定アイテムに対して「カンパ（寄付）」など、一時的にゴールドを支払う援助システムを導入する可能性がある

### 4.3. フェイズとターンの進行
ターンは以下のフェイズで構成され、全プレイヤーが「同時に」進行するリアルタイム・フェイズ制を採用する。（チェーンブロック、スタック的な解決は行わず、順次処理または同時解決）

1. **スタートフェイズ**: 
   - 全プレイヤーは古い手札を全て捨札に送り、新たに山札から5枚ドローする。
   - 各プレイヤーのエナジー(行動力)が「3」回復する（最大値5）。
   - キャラクターに付与されている**「防御値(アーマー/ブロック)」はこの時点で0にリセット**される。
   - 状態異常・バフ・デバフなど、発動ターン数を持つ効果のカウントダウン処理が行われる。

2. **メインフェイズ**: 
   - 各プレイヤーは自分のエナジーを消費し、順不同・同時進行でカードを使用したり、武器を装備したりする。
   - **ターゲット予約**: 各プレイヤーはこのフェイズの間に、次に行う攻撃の**「ターゲット(敵)」**を指定する。

3. **バトルフェイズ**: 
   - **プレイヤー側（味方連携）の攻撃**: 
     各プレイヤー自身(武器装備時) および その支配下にあるユニット(最大3体) が、**左から順番に**メインフェイズで各プレイヤーが指定した単一ターゲットに向かって攻撃を行う。
     *(特記事項)*: ユニットやプレイヤーの攻撃処理中に指定ターゲットの敵が死亡した場合でも、攻撃対象の切り替えは行われない。残りの攻撃は空振り＝「無駄打ち」となる。全体攻撃やランダム対象攻撃などの特殊指向は個別のカード効果として定義可能。
   - **敵側連携の攻撃**: 
     生き残っている敵が、前の回の行動で可視化(予告)していた意志（攻撃する、等）を一斉に実行する。
     *(特記事項)*: 敵の攻撃は常に**「全体攻撃」**として扱われ、場にいる生存中の全てのプレイヤーおよび全てのユニット向けに対してダメージが計算される。個々が各自身に防御値がある場合、ダメージ量は個別に軽減される。

## 5. カードシステム要件
- **プレイヤー自身**: デフォルトで存在。体力(HP)を持つ主体。装備を付けることで攻撃行動が可能。
- **ユニットカード**: プレイヤーが召喚し場に残る味方(最大3体まで)。個別の体力を持つ。
- **武器（アイテム）**: プレイヤー本体、またはユニットに対して**「1つまで」**装備可能。
  - すでに装備・武器を持っている対象に新しい装備を使用した場合は、古い装備が**上書き(破棄)**される。
  - 一部の武器には**「耐久値」**が存在し、攻撃を一定回数行うなどの条件で破壊される。
- **イベント（スキル）カード**: 使用時に即時効果を発揮して捨て札へ行くもの。

## 6. 報酬システム
- 敵撃破時の報酬（「ゴールド」をどれだけ貰えるか、「アイテム」をいくつ貰えるか、「カード」パックをいくつ出来るか等の基準・枠組み）は、**パーティ全員に対して平等**に与えられる。
- ただし、その**中身（提示されるカードの選択肢やアイテムの種類）**は、各プレイヤーごとに別々にランダム抽選される。
  - *(例)*: 報酬として「レジェンダリー」レアリティのカードの「パック」が提供された場合、Aさんの選択肢とBさんの選択肢には異なるパターンのレジェンダリーカードが提示され、全員がそれぞれ自分のカードを獲得する。
